# NIHONGO.GO — 产品设计方案 FINAL

> 一款专为日本 Solo Trip 设计的极简像素风日语短句学习 PWA

---

## 一、产品概览

| 项目 | 内容 |
|------|------|
| 产品名称 | NIHONGO.GO |
| 产品形态 | PWA（Safari 添加到 iPhone 主屏幕，等同原生 App） |
| 目标用户 | 本人，N5 水平，备战日本独旅 |
| 核心目标 | 能在真实旅行场景中自由开口对话 |
| 学习时长 | 每日专注 15–30 分钟 |

## 二、内容规划

### 2.1 场景划分

| 场景 ID | 场景名 | 覆盖话题 | 预计句数 |
|---------|--------|----------|----------|
| S1 | ✈️ AIRPORT | 值机、托运、安检、问路、换乘、登机 | ~80句 |
| S2 | 🏨 HOTEL | 入住、退房、问服务、投诉、设施、叫车 | ~80句 |
| S3 | 🍶 IZAKAYA | 点单、询问菜品、过敏确认、干杯、加单、结账 | ~80句 |
| S4 | 🛍️ SHOPPING | 找商品、询问尺码、试穿、付款、退换、免税 | ~80句 |
| **合计** | | | **~320句** |

### 2.2 内容结构：场景 → 小课 → 对话短句

每个场景按实际旅行节点拆分为若干**小课**，小课顺序解锁，每课 5–8 个对话对：

```
S1 AIRPORT
├── Lesson 01  办理值机
├── Lesson 02  行李托运
├── Lesson 03  安检过关
├── Lesson 04  问路 & 找登机口
├── Lesson 05  登机 & 机上
└── Lesson 06  入境 & 海关
```

### 2.3 内容形式：完整对话对（有问有答）

每个知识点以**完整对话对**呈现，而非孤立单句：

```
场景：机场问路

[你说]   すみません、搭乗ゲートはどこですか？
         すみません、とうじょうゲートはどこですか？
         请问，登机口在哪里？

[对方说] A番ゲートは右へ行ってください。
         Aばんゲートはみぎへいってください。
         A号登机口请往右走。
```

### 2.4 每条内容包含的字段

| 字段 | 说明 |
|------|------|
| 日文原文 | 汉字书写 |
| 振り仮名 | 每个汉字正上方标注假名读音 |
| 中文翻译 | 自然口语翻译，非直译 |
| 音频 | 优先真人录音，无则 TTS（Web Speech API） |
| 对话角色 | 标注「你说」或「对方说」 |
| 场景/小课 | 所属 S1–S4 及 Lesson 编号 |
| 核心标记 | `isCore` 字段，后台人工标注高频句 |

> 难度不做 JLPT 分级，以「旅行实用度」为唯一筛选标准。

### 2.5 单词卡系统

每个场景维护独立关键词库，不从句子中自动抽取。单词学习走简化流程：**听 → 读 → 写 → 验证**（不需要「说」步骤）。单词与句子在复习和练习中混合出现。

## 三、学习流程设计

### 3.1 新内容学习：五步法（听→说→读→写→验证）

每次学习一个对话对，按以下顺序**强制顺序推进**，五步全部完成且「说」与「验证」均通过，才进入已学会内容库：

```
┌──────────────────────────────────────────────┐
│  STEP 1 / 听                                 │
│  播放音频，屏幕只显示中文翻译                 │
│  可重复播放，建立语音印象                     │
├──────────────────────────────────────────────┤
│  STEP 2 / 说                                 │
│  显示振り仮名全文，跟读后录音                 │
│  → 优先调用 Azure 发音评估打分               │
│  → Azure 不可用时降级为录音回放 + 手动确认   │
│  达到及格分（或手动确认）才能进入下一步       │
├──────────────────────────────────────────────┤
│  STEP 3 / 读                                 │
│  显示完整对话（汉字 + 振り仮名 + 译文）      │
│  阅读理解句子结构，可展开语法注释             │
├──────────────────────────────────────────────┤
│  STEP 4 / 写                                 │
│  遮住日文，只显示中文 + 可选提示（假名/首词）│
│  用日语输入法手动输入完整句子                 │
│  提交后逐字对比高亮差异                       │
├──────────────────────────────────────────────┤
│  STEP 5 / 验证                               │
│  再次用中文提示，完整输入标准日语             │
│  无任何提示，严格匹配标准答案                 │
│  通过 → 句子进入已学会内容库 ✓              │
│  未通过 → 返回 STEP 4 重试                   │
└──────────────────────────────────────────────┘
```

### 3.2 每日掌握检测（Daily Check）

每次学习结束后自动触发：

- 从**今日学习的内容**中随机抽取 5–8 条
- 题型混合：
  - 听音选义（听音频，四选一中文）
  - 中译日输入（看中文，输入日文）
  - 对话补全（给出上句，输入下句）
- 结果判定：
  - ≥ 80% → 今日内容标记「已掌握」，进入 SRS 复习队列
  - < 80% → 标记「待复习」，明日优先推送

## 四、进度与复习系统

### 4.1 掌握度状态

| 状态 | 含义 |
|------|------|
| `⬜ 未学` | 从未接触 |
| `🟨 学过` | 完成五步但未通过 Daily Check |
| `🟦 掌握` | 通过 Daily Check，进入 SRS |
| `🟩 熟练` | 连续 2 次复习评为「会」 |
| `⭐ 巩固` | SRS 间隔达 8 天以上 |

### 4.2 SRS 复习规则

复习时每题采用**三档自评**：

| 自评 | 含义 | 下次间隔 |
|------|------|---------|
| `不会` | 完全想不起来 | 重置为 1 天 |
| `模糊` | 有印象但不确定 | 间隔不变 |
| `会` | 流畅想起 | 间隔 × 2 |

间隔递进：1天 → 2天 → 4天 → 8天 → 16天

复习题型为混合：**中文→回忆日语** 与 **日语→回忆中文**，句子优先，混入少量单词。轻量题自评即可进入下一轮，强化题（连续答错）需要完整日语输入验证。

### 4.3 首页每日看板

```
📬 今日复习：N 句     🆕 今日新课：N 句
整体进度 ████████░░░░  已掌握 128 / 320
```

## 五、练习模式（独立场景）

### 5.1 进入设置

1. 选择**场景范围**：全部 / S1 / S2 / S3 / S4
2. 选择**题型**：句子 / 单词 / 混合
3. 选择**数量 N**：5 / 10 / 20 / 自定义

> 只从**已学会内容库**中抽题，未完成五步的内容不进入练习池。

### 5.2 练习规则

- 题面固定为**中文提示**，用日语输入法输入完整日文
- 判题为**严格匹配标准答案**，不做 AI 宽松判题
- 提交后逐字对比，高亮差异，标记 ✓ / △ / ✗

### 5.3 错题重练机制

- 第一轮结束后自动进入**错题重练轮**
- 只包含本次答错的题目
- 持续循环，直到本次所有错题全部答对才结束
- 练习结果**不反向影响 SRS 复习队列**

## 六、出发模式（✈️ 临行冲刺）

### 6.1 概念

出发前 1–3 天开启的**高强度冲刺模式**，只练最重要的内容。

### 6.2 内容池来源

- 学习/复习页随手**收藏**的句子
- 后台人工标注 `isCore = true` 的高频核心句

### 6.3 训练方式

- 题型固定为**中文 → 日语输出**
- 句子为主，少量单词辅助
- 同样有错题重练机制

### 6.4 收藏入口

收藏按钮常驻于**学习页**和**复习页**右上角，随时加入出发模式内容池。

## 八、UI 设计规范

### 8.1 视觉风格

**Game Boy 复古像素风**，参考调性：暖、克制、复古、护眼

| 元素 | 规格 |
|------|------|
| 背景色 | `#E8E4D0`（奶油米白） |
| 主文字色 | `#3D3A2E`（深橄榄棕） |
| 强调色 | `#6B7C3E`（苔藓绿） |
| 错误色 | `#8B3A3A`（暗砖红） |
| 成功色 | `#3A6B4A`（深林绿） |
| 英文/数字字体 | Press Start 2P（像素点阵） |
| 日文字体 | Noto Sans JP（清晰易读） |
| 像素装饰 | 像素爱心 ♥、像素实线边框、扫描线过渡动画 |

### 8.2 交互细节

- **按钮点击**：像素风按压动效（向下偏移 2px + 阴影消失）
- **答对反馈**：屏幕边缘绿色像素闪烁
- **答错反馈**：屏幕轻微震动 + 红色像素闪烁
- **场景切换**：像素扫描线横扫过渡动画
- **进度条**：方块堆叠样式（非普通圆角条）
- **发音评分**：像素风仪表盘展示 Azure 评分

### 8.3 振り仮名渲染

使用 HTML `<ruby>` 标签实现汉字上方假名标注：

```html
<ruby>搭乗<rt>とうじょう</rt></ruby>ゲートは
<ruby>右<rt>みぎ</rt></ruby>へ<ruby>行<rt>い</rt></ruby>ってください。
```

## 九、技术架构

### 9.1 技术栈

| 层级 | 方案 | 费用 |
|------|------|------|
| 框架 | Next.js 14（App Router） | 免费 |
| 部署 | Vercel | 免费 |
| 数据层 | 静态 JSON 文件（内嵌代码仓库） | 免费 |
| 音频（TTS） | Web Speech API（浏览器内置日语） | 免费 |
| 音频（ASR/发音评估） | Azure Speech Service F0 免费层 | 免费* |
| Azure 代理 | Vercel Edge Function（Key 藏服务端） | 免费 |
| 进度存储 | localStorage | 免费 |
| PWA 配置 | next-pwa 插件 | 免费 |
| 字体 | Google Fonts（Press Start 2P + Noto Sans JP） | 免费 |

> *Azure Speech F0 免费层：STT 5小时/月，TTS 50万字符/月。个人每日30分钟练习，月用量约 15–20 分钟，远低于免费额度上限。

### 9.2 Azure 调用架构

```
iPhone Safari
    ↓ 录音 Blob
Vercel Edge Function（/api/speech）
    ↓ 携带 Azure Key（环境变量，客户端不可见）
Azure Speech Service
    ↓ 发音评估 JSON
Vercel Edge Function
    ↓ 返回评分结果
iPhone Safari 展示像素评分
```

## 十一、不做的事（边界约定）

- ❌ 不做用户账号 / 云同步（localStorage 已足够）
- ❌ 不做社交功能

- ❌ 不做付费订阅功能
